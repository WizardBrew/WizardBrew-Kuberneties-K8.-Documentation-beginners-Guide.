<h2>Cluster Information</h2> <ul> <li><code>kubectl get nodes</code> → Shows number of nodes in the cluster</li> <li><code>kubectl cluster-info</code> → Displays cluster information</li> <li><code>kubectl config get-contexts</code> → Shows config files to connect from different remote locations</li> </ul>

<h2>Namespaces</h2> <ul> <li><code>kubectl get namespace</code> or <code>kubectl get ns</code> → Lists namespaces in the cluster</li> <li><code>kubectl create ns batch39</code> → Creates a new namespace named batch39</li> <li><code>kubectl create -f <a href="https://phatan.yaml">phatan.yaml</a></code> → Creates a namespace using YAML configuration</li> <li><code>kubectl apply -f <a href="https://pathan.yaml">pathan.yaml</a></code> → Creates or updates resources using YAML (recommended)</li> </ul>

<h2>Pods</h2> <ul> <li><code>kubectl get pods</code> or <code>kubectl get po</code> → Lists all pods</li> <li><code>kubectl get pods -o wide</code> → Lists pods with extended information</li> <li><code>kubectl get pods -l app=nginx</code> → Lists pods with label <code>app=nginx</code></li> <li><code>kubectl get pods -n default</code> → Lists pods in the <code>default</code> namespace</li> <li><code>kubectl get pods -n my-nginx-podspace</code> → Lists pods in the <code>my-nginx-podspace</code> namespace</li> <li><code>kubectl get pods --all-namespaces | grep nginx</code> → Filters pods across all namespaces for nginx</li> </ul>

<h2>ReplicaSets</h2> <ul> <li><code>kubectl get replicaset</code> or <code>kubectl get rs</code> → Lists ReplicaSets</li> </ul>

<h2>Services</h2> <ul> <li><code>kubectl get service</code> or <code>kubectl get svc</code> → Lists services</li> <li><code>kubectl get svc -n default</code> → Lists services in the <code>default</code> namespace</li> <li><code>kubectl edit svc nginx -n <namespace></code> → Edits the service named nginx in the specified namespace</li> </ul>

<h2>Resource Details and Deletion</h2> <ul> <li><code>kubectl describe <resource></code> → Describes a resource (e.g., node, pod, rs, svc)</li> <li><code>kubectl delete <resource></code> → Deletes a resource (e.g., node, pod, rs, svc)</li> </ul>

<h2>Running and Exposing Applications</h2> <ul> <li><code>kubectl run nginx --image=nginx:1.6</code> → Runs nginx with version 1.6</li> <li><code>kubectl run nginx --image=nginx:latest</code> → Runs nginx with the latest version</li> <li><code>kubectl expose pod nginx --port=80 --target-port=80 --type=LoadBalancer -n my-nginx-podspace</code> → Exposes the nginx pod</li> <li><code>kubectl expose deployment nginx --port=80 --target-port=80 --type=LoadBalancer -n my-nginx-podspace</code> → Exposes the nginx deployment</li> </ul>

<p class="note">Note: <code>-n</code> flag specifies the namespace.</p>
