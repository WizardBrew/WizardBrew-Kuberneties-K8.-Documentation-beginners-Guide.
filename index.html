Kubernetes Documentation

Cluster Information

  • kubectl get nodes → Shows number of nodes in the cluster
  • kubectl cluster-info → Displays cluster information
  • kubectl config get-contexts → Shows config files to connect from different remote locations

Namespaces

  • kubectl get namespace or kubectl get ns → Lists namespaces in the cluster
  • kubectl create ns batch39 → Creates a new namespace named batch39
  • kubectl create -f phatan.yaml → Creates a namespace using YAML configuration
  • kubectl apply -f pathan.yaml → Creates or updates resources using YAML (recommended)

Pods

  • kubectl get pods or kubectl get po → Lists all pods
  • kubectl get pods -o wide → Lists pods with extended information
  • kubectl get pods -l app=nginx → Lists pods with label app=nginx
  • kubectl get pods -n default → Lists pods in the default namespace
  • kubectl get pods -n my-nginx-podspace → Lists pods in the my-nginx-podspace namespace
  • kubectl get pods --all-namespaces | grep nginx → Filters pods across all namespaces for nginx

ReplicaSets

  • kubectl get replicaset or kubectl get rs → Lists ReplicaSets

Services

  • kubectl get service or kubectl get svc → Lists services
  • kubectl get svc -n default → Lists services in the default namespace
  • kubectl edit svc nginx -n <namespace> → Edits the service named nginx in the specified namespace

Resource Details and Deletion

  • kubectl describe <resource> → Describes a resource (e.g., node, pod, rs, svc)
  • kubectl delete <resource> → Deletes a resource (e.g., node, pod, rs, svc)

Running and Exposing Applications

  • kubectl run nginx --image=nginx:1.6 → Runs nginx with version 1.6
  • kubectl run nginx --image=nginx:latest → Runs nginx with the latest version
  • kubectl expose pod nginx --port=80 --target-port=80 --type=LoadBalancer -n my-nginx-podspace → Exposes the nginx pod
  • kubectl expose deployment nginx --port=80 --target-port=80 --type=LoadBalancer -n my-nginx-podspace → Exposes the nginx deployment

Note: -n flag specifies the namespace.
