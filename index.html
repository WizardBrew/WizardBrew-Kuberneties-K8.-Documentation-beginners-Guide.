📘 Kubernetes Beginner Guide

🔧 Cluster Basics

  • kubectl get nodes — Shows number of nodes in the cluster
  • kubectl cluster-info — Displays cluster information
  • kubectl config get-contexts — Lists config files for remote access

📂 Namespaces

  • kubectl get namespace or kubectl get ns — Lists namespaces
  • kubectl create ns batch39 — Creates a namespace named batch39
  • kubectl create -f phatan.yaml — Creates namespace using YAML
  • kubectl apply -f pathan.yaml — Recommended for creating/updating resources

📦 Pods

  • kubectl get pods or kubectl get po — Lists all pods
  • kubectl get pods -o wide — Shows extended pod info
  • kubectl get pods -l app=nginx — Filters pods with label app=nginx
  • kubectl get pods -n default — Pods in default namespace
  • kubectl get pods -n my-nginx-podspace — Pods in custom namespace
  • kubectl get pods --all-namespaces | grep nginx — Search nginx pods across namespaces

📊 ReplicaSets

  • kubectl get replicaset or kubectl get rs — Lists ReplicaSets

🌐 Services

  • kubectl get service or kubectl get svc — Lists services
  • kubectl get svc -n default — Services in default namespace
  • kubectl edit svc nginx -n <namespace> — Edit service type

🧹 Resource Management

  • kubectl describe <resource> — Describe node, pod, rs, svc
  • kubectl delete <resource> — Delete node, pod, rs, svc

🚀 Running & Exposing Applications

  • kubectl run nginx --image=nginx:1.6 — Run nginx v1.6
  • kubectl run nginx --image=nginx:latest — Run latest nginx
  • kubectl expose pod nginx --port=80 --target-port=80 --type=LoadBalancer -n my-nginx-podspace — Expose nginx pod
  • kubectl expose deployment nginx --port=80 --target-port=80 --type=LoadBalancer -n my-nginx-podspace — Expose nginx deployment

Note: -n specifies the namespace.
